ЛЕКЦИЯ 3.1. ФУНКЦИИ

== Зачем нужны функции ==

Функция — это именованный блок кода, который можно вызывать многократно. Функции избавляют от дублирования, делают код читаемым и позволяют разбивать задачу на понятные шаги. Правило курса: если один и тот же фрагмент кода встречается дважды — выносите его в функцию.

== Определение и вызов ==

    def greet(name):
        print(f"Привет, {name}!")

    greet("Анна")   # Привет, Анна!
    greet("Борис")  # Привет, Борис!

Ключевое слово def начинает определение, в скобках перечисляются параметры, тело функции пишется с отступом. Функция выполняется только при вызове — само определение ничего не печатает.

== return: вернуть, а не напечатать ==

Самая частая путаница у новичков — разница между print и return. print показывает значение на экране, return возвращает его в программу, чтобы результат можно было использовать дальше:

    def add(a, b):
        return a + b

    result = add(2, 3)   # result == 5
    print(add(result, 10))  # 15

Функция без return возвращает специальное значение None. Если вы написали функцию с print внутри, а потом пытаетесь сложить её результат с числом — получите TypeError, потому что складываете число с None.

После выполнения return функция немедленно завершается: код ниже не выполнится.

== Аргументы по умолчанию и именованные аргументы ==

    def power(base, exponent=2):
        return base ** exponent

    power(5)              # 25 — используется значение по умолчанию
    power(5, 3)           # 125
    power(exponent=3, base=2)  # 8 — именованные аргументы можно менять местами

Параметры со значениями по умолчанию должны идти после обычных параметров.

== Область видимости ==

Переменные, созданные внутри функции, — локальные: снаружи они не видны и исчезают после завершения вызова.

    def calc():
        temp = 100   # локальная переменная

    calc()
    print(temp)      # NameError: temp не существует

Глобальные переменные (созданные на верхнем уровне файла) внутри функции читать можно, но менять без ключевого слова global нельзя. В рамках курса мы придерживаемся правила: функции получают данные через параметры и отдают через return, глобальные переменные не изменяем.

== Докстринги ==

Первой строкой функции принято писать документацию — докстринг в тройных кавычках:

    def add(a, b):
        """Возвращает сумму двух чисел."""
        return a + b

В домашних заданиях начиная с модуля 3 докстринг обязателен для каждой функции — это входит в критерии оценки (см. методичку по оформлению домашних заданий).

== Итоги ==

Функция определяется через def и выполняется при вызове; return возвращает значение, print лишь отображает; функция без return возвращает None; локальные переменные живут только внутри функции; докстринги обязательны в ДЗ с модуля 3.
