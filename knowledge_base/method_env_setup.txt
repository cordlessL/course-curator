МЕТОДИЧКА. УСТАНОВКА И НАСТРОЙКА РАБОЧЕГО ОКРУЖЕНИЯ

== Шаг 1. Установка Python ==

На курсе используется Python версии 3.12 или новее.

Windows:
1. Скачайте установщик с официального сайта python.org, раздел Downloads.
2. Запустите установщик и ОБЯЗАТЕЛЬНО поставьте галочку «Add Python to PATH» на первом экране. Пропуск этой галочки — причина 90% проблем «python не является внутренней или внешней командой».
3. Нажмите Install Now и дождитесь завершения.

macOS:
Рекомендуемый способ — установка через Homebrew командой brew install python. Альтернатива — установщик с python.org.

Проверка установки: откройте терминал (Windows — PowerShell, macOS — Terminal) и выполните:

    python --version

Должна отобразиться версия, например Python 3.12.4. На macOS команда может называться python3.

== Шаг 2. Установка VS Code ==

1. Скачайте Visual Studio Code с официального сайта code.visualstudio.com.
2. Установите с настройками по умолчанию.
3. Откройте VS Code, перейдите во вкладку Extensions (Ctrl+Shift+X) и установите расширение Python от Microsoft.
4. Там же установите расширение Ruff — оно будет подсвечивать ошибки оформления кода по стандартам курса.

== Шаг 3. Настройка отступов ==

Откройте настройки (Ctrl+,) и проверьте параметры:
- Editor: Tab Size = 4
- Editor: Insert Spaces = включено

Это гарантирует, что клавиша Tab вставляет 4 пробела — стандарт PEP 8 и требование курса.

== Шаг 4. Первый запуск программы ==

1. Создайте папку курса, например C:\python_course или ~/python_course.
2. В VS Code откройте эту папку: File -> Open Folder.
3. Создайте файл hello.py с одной строкой: print("Привет, курс!").
4. Запустите файл кнопкой Run (треугольник в правом верхнем углу) или в терминале командой python hello.py.

Если в терминале появился текст «Привет, курс!» — окружение готово.

== Частые проблемы ==

«python не распознан как команда» (Windows): Python установлен без галочки Add to PATH. Решение — переустановить Python с этой галочкой либо использовать команду py вместо python.

VS Code не видит Python: нажмите Ctrl+Shift+P, введите «Python: Select Interpreter» и выберите установленную версию 3.12.

Кириллица отображается неправильно при работе с файлами: всегда указывайте encoding="utf-8" в функции open (подробнее — лекция 5.1).

== Требования к окружению для сдачи ДЗ ==

Домашние задания принимаются в виде .py-файлов, написанных в любом редакторе, однако проверяющий скрипт запускает их на Python 3.12. Использование конструкций, недоступных в 3.12, приведёт к ошибке проверки. Jupyter-ноутбуки принимаются только в итоговом проекте.
