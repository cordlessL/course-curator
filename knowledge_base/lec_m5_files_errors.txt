ЛЕКЦИЯ 5.1. ФАЙЛЫ И ИСКЛЮЧЕНИЯ

== Чтение файла ==

Стандартный способ работы с файлами — контекстный менеджер with, который гарантирует закрытие файла даже при ошибке:

    with open("notes.txt", "r", encoding="utf-8") as f:
        text = f.read()

Режимы открытия: "r" — чтение (файл должен существовать), "w" — запись с перезаписью содержимого, "a" — дозапись в конец. Параметр encoding="utf-8" на курсе обязателен всегда: без него на Windows русский текст может прочитаться «кракозябрами».

Построчное чтение:

    with open("notes.txt", encoding="utf-8") as f:
        for line in f:
            print(line.strip())   # strip убирает перевод строки

== Запись в файл ==

    with open("result.txt", "w", encoding="utf-8") as f:
        f.write("первая строка\n")
        f.write("вторая строка\n")

Внимание: режим "w" стирает прежнее содержимое файла без предупреждения. Если нужно добавить данные к существующим — используйте режим "a".

== Что такое исключение ==

Исключение — это ошибка, возникающая во время выполнения программы: деление на ноль (ZeroDivisionError), преобразование «не числа» в int (ValueError), открытие несуществующего файла (FileNotFoundError), обращение к несуществующему ключу словаря (KeyError). Необработанное исключение останавливает программу и печатает трассировку (traceback). Читать traceback нужно снизу вверх: в последней строке — тип и описание ошибки, выше — место в коде.

== Обработка try/except ==

    try:
        raw = input("Введите число: ")
        number = int(raw)
        print(100 / number)
    except ValueError:
        print("Это не число")
    except ZeroDivisionError:
        print("На ноль делить нельзя")

Правила курса по обработке исключений:
1. Ловите конкретные типы ошибок, а не голый except: — «глухая» обработка прячет настоящие баги.
2. Блок try должен быть коротким: оборачивайте только ту строку, которая может упасть.
3. Не используйте try/except для управления обычной логикой программы, если можно обойтись условием.

Дополнительно существуют блоки else (выполняется, если исключения не было) и finally (выполняется всегда — удобно для освобождения ресурсов).

== Мини-паттерн: надёжный ввод числа ==

    while True:
        try:
            age = int(input("Возраст: "))
            break
        except ValueError:
            print("Введите целое число, например 25")

Этот паттерн — цикл с try/except — вы будете использовать в итоговом проекте курса для проверки пользовательского ввода.

== Итоги ==

Файлы открываем только через with и с encoding="utf-8"; режим "w" перезаписывает файл; исключения обрабатываем через try/except с указанием конкретного типа ошибки; traceback читается снизу вверх; «голый» except на курсе считается ошибкой оформления.
