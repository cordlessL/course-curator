МЕТОДИЧКА. ПРАВИЛА ОФОРМЛЕНИЯ И СДАЧИ ДОМАШНИХ ЗАДАНИЙ

== Формат сдачи ==

1. Каждое домашнее задание сдаётся одним .py-файлом через личный кабинет LMS, раздел «Задания».
2. Имя файла: hw<номер модуля>_<фамилия латиницей>.py, например hw2_ivanova.py.
3. Точные дедлайны по каждому заданию указаны в LMS. Куратор и материалы курса не являются источником дедлайнов — всегда сверяйтесь с личным кабинетом.
4. Работы, сданные после дедлайна, проверяются, но максимальный балл снижается на 20%. Задания, сданные позже чем через 7 дней после дедлайна, не оцениваются.

== Требования к коду ==

1. Отступы — 4 пробела, табуляция не допускается.
2. Имена переменных и функций — латиницей в snake_case, осмысленные: total_price, а не tp или x1.
3. Начиная с модуля 3 каждая функция должна иметь докстринг — краткое описание в тройных кавычках.
4. Магические числа выносите в переменные с понятными именами.
5. Код должен запускаться без ошибок командой python имя_файла.py на Python 3.12. Работа, которая падает при запуске, получает 0 баллов независимо от содержания.

== Критерии оценивания ==

Каждое задание оценивается из 10 баллов:
- 6 баллов — корректность: программа решает задачу на всех проверочных примерах;
- 2 балла — оформление: имена, отступы, докстринги;
- 2 балла — читаемость и структура: отсутствие дублирования, разумное разбиение на функции.

Оценки выставляет проверяющий преподаватель в LMS в течение 5 рабочих дней после дедлайна. Куратор курса оценки не выставляет и не изменяет; вопросы о баллах решаются через апелляцию.

== Апелляция ==

Если вы не согласны с оценкой, в течение 3 дней после её публикации напишите преподавателю через LMS кнопкой «Оспорить оценку» с пояснением, с какими пунктами проверки вы не согласны. Ответ по апелляции приходит в течение 5 рабочих дней.

== Академическая честность ==

Разрешается обсуждать подходы к решению с другими студентами и пользоваться документацией. Запрещается сдавать чужой код или код, полностью сгенерированный нейросетью, без самостоятельной проработки. При обнаружении идентичных работ обе получают 0 баллов. Использовать ИИ-помощники для объяснения непонятных тем — можно и даже полезно; выдавать их решения за свои — нельзя.

== Пересдачи ==

Каждое задание можно пересдать один раз до окончания модуля. При пересдаче учитывается последняя отправленная версия. Итоговый проект пересдаче не подлежит.
